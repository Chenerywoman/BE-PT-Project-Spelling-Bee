Spelling Bee: API routes

api/words

  • GET /api/words
  • Returns an object containing an array of word objects which each contain the word and a categories object with four arrays: prefixes, suffixes, medials & homophones.
  • POST /api/words
  • Posts a word object to the database. This requires a JSON body with the following structure: {"word": "pineapple", "categories": {"prefixes" : ["pine"], "suffixes": ["le"], "medials: [], "homophones": []}}

api/words/prefixes

  • GET /api/words/prefixes
  • Returns an object with an array of prefixes.
  • GET /api/words/prefixes?prefix={prefix}
  • With a prefix query e.g /api/words/prefixes?prefix=anti returns an object with an array of word objects where the prefixes array contains the query string.

api/words/suffixes

  • GET /api/words/suffixes
  • Returns an object with an array of suffixes.
  • GET /api/words/suffixes?suffix={suffix}
  • With a suffix query e.g /api/words/suffixes?suffix=ly returns an object with an array of word objects where the suffixes array contains the query string.

api/words/medials

  • GET /api/words/medials
  • Returns an object with an array of medials.
  • GET /api/words/medials?medial={}
  • With a medial query e.g /api/words/medials?medial=sc returns an object with an array of word objects where the medials array contains the query string.

api/words/homphones

  • GET /api/words/homphones
  • Returns an object with an array of homophones.
  • GET /api/words/homophones?homophone={}
  • With a homophone query e.g /api/words/homophones?homophone=break returns an object with an array of word objects where the homophones array contains the query string.

api/words/freestyle

  • GET /api/words/freestyle
  • Returns an object with an array of words where all four categories arrays (prefixes, suffixes, medials & homophones) are empty.
